Introduction to NodeJS

Although NodeJS is one of the youngest among the most popular web technologies, it has the largest Open-Source community and the largest Repository of Libraries and Packages.

how Node JS works as a backend?

The Node JS Platform does not follow Request/Response Multi-Threaded Stateless Model. It follows Single Thread concept with Event Loop Model, to handle multiple concurrent clients. The Node JS Processing model is mainly based on Javascript Callback mechanism. It utilizes less resources as compared to multi-threaded model. It is a good choice for building modern solutions based on Micro-services, event queues, jobs, and WebSockets.

How it handle multiple requests from clients?

Node JS uses two concepts - Asynchronous & Non-Blocking I/O. A non-blocking call initiates the operation, leaves it for the OS to complete returning immediately without any results. Asynchronous operations use callback functions, and return results by creating a wait mechanism in a non-blocking operation. Modern approach uses Async/await technique, to avoid callback hells and handle promises (that involved function chaining).

what is libuv?

A multiplatform Node JS library, built in C language, uses system kernel and implements multiple threads, with the help of which Node JS uses multithreading indirectly. It highlights the following -

  1. Asynchronous I/O
  2. File system events
  3. Child processes
  4. Signal handling
  5. Threading and synchronization primitives.
